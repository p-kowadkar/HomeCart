# Decision Log: Cartographer

This document tracks the core design decisions made during the brainstorming phase for the Cartographer project.

| Date | Decision | Alternatives Considered | Rationale |
| :--- | :--- | :--- | :--- |
| 2026-05-09 | **Onboarding: Profile First** | Utility First / Progressive Discovery | Ensures every result is culturally anchored from the very first interaction. |
| 2026-05-09 | **Auth: Social-Only** | Magic Link / Traditional / Ghost Profile | Keeps the mandatory onboarding flow fast, secure, and frictionless. |
| 2026-05-09 | **Cultural Specificity: National Level** | Regional Level / Ingredient-Driven | Best balance for MVP; provides immediate value without over-complicating onboarding. |
| 2026-05-09 | **Home Screen: Unified Feed** | Camera-only / Search-only / List-only | Acts as a companion, balancing "Survival Mode" (Scan) and "Planning Mode" (List). |
| 2026-05-09 | **Search Logic: Match Score (%)** | Functional Labels / Recipe Validation | Provides a clear "Why" behind recommendations with actionable AI tips. |
| 2026-05-09 | **Scanner: Magic Lens (AR)** | Snapshot Mode / Barcode Scanner | Maximizes speed and confidence in-store; minimizes "aisle awkwardness." |
| 2026-05-09 | **List Gen: Recipe Importer** | Essential Templates / Aisle Builder | Lowers cognitive load; focuses on the desired *meal* rather than grocery chores. |
| 2026-05-09 | **Store Discovery: Store Persona Map** | Price Comparison / General Map | Helps users build a mental model of the US retail landscape (e.g., "Aldi" vs "Whole Foods"). |
| 2026-05-09 | **Monetization: Free (Scale First)** | Affiliate / Premium / Partner Stores | Prioritizes user growth and engine accuracy over immediate revenue. |
